我的博客配置经历

发表于 2020-05-18

这篇文章是用来记录我配置个人博客的经历。主要使用了github pages+Hexo+NexT的组合。

第一步 搭建本地博客开发环境

个人博客的开发需要用到Hexo，在此之前还需要保证两个东西可用，分别是：

  1. Nodejs
  2. git

没有记录Nodejs和git的安装，因为这两个玩意实在是太基本了，经常会用到，而Hexo就不怎么必要了，除了Hexo还有其他方式配置个人博客，例如vuepress，Hugo等等。

1. 通过npm全局安装Hexo

2. 创建本地项目

这条命令执行以后，会在当前目录创建一个blog-creator文件，并将最基础的项目文件clone到这个文件夹下。

最基本的文件目录有:

  • _config.yml：博客的配置文件
  • package.json：项目依赖的各种插件，以及版本号
  • scaffolds：存储了各种模板文件，文件夹下默认包含了三个文件:
    • draft.md：草稿文件的模板文件，当通过命令生成草稿文件的时候，新文件中会包含此文件中定义的各种信息
    • page.md：页面文件的模板文件，作用同上。
    • post.md：博客文件的模板文件。
  • source：存储各种页面文件，默认情况下，此文件夹下包含有一个文件夹。
    • _post：主要存储.md形式的博客文件，当项目生成静态页面时，文件下的博客文件会生成对应的页面。
  • themes：主题文件夹，存储博客的主题文件，有一个默认主题。
    • landscape：默认的博客主题。
